Searching for Books


Book Identifiers

Google utilizes 3 different kinds of IDs:

  1. Volume IDs
  2. Bookshelf IDs
  3. User IDs

The focus of this tutorial will be to use the volume ID to extract information specific to the book edition. Google uses this internal reference code, a unique string that appears to be a mix of random alphabets and numbers with no real-world relevance, to catalog their library database.

There are many different methods, but the easiest way to obtain this ID code is to head over to books.google.com and select the book of choice.

The volume ID can be found in the id parameter of the URL in the browser address bar. For example:

https://books.google.com/books?id=CxxGPgAACAAJ&dq=harry+potter+and+the+sorcerer%27s+stone&hl . . .

You can now use this id string to look up the data that makeup this book object, simply by appending the id value to the end of this URL http://www.google.com/books/feeds/volumes/ like so:


http://www.google.com/books/feeds/volumes/CxxGPgAACAAJ

The output?

Great. What we have here is still not particularly useful, is it? You can make out a few key words here and there, but that's a poor way of handling the data. What we really want to do is create an API request and choose among the number of methods provided to extract the information that we really want. This is where your API key comes into play.
