Accessing Google Books API


Authentication Modes

Google grants you access permission on the condition that it can identify the appllication requesting the data. There are 2 ways to go about this:

  • API Key: Used for requesting public data, such as page count or book reviews, and can be substituted with an OAuth credential.
  • OAuth 2.0: Used for interacting with private user data, such as an account's non-public bookshelf contents.

Getting Your API Key

The first step to retrieving the API key is to register your application with Google Developers Console. You will need a Google account to create a project before activating the Books API on the console.

There are 4 types of API keys: server, browser, iOS, and Android. Go ahead and select the browser key type, which is used on client applications, when prompted.

It is very important that this key is safeguarded appropriately, as it will be used in all your future data requests.
